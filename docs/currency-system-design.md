# 재화 & 성장 시스템 설계서

## 1. 현재 상태 (AS-IS)

| 항목 | 내용 |
|------|------|
| 재화 | Bagel Coin (단일) |
| 수급 | 게임 완료 시 `baseReward + score x scoreRewardMultiplier` |
| 소모 | 미니게임 언락 (`unlockCost`) |
| 초기 지급 | 30코인 |
| 문제점 | 수급=게임클리어, 소모=언락 한 가지 경로뿐. 모든 게임 언락 후 코인 쓸 곳 없음 |

---

## 2. 목표 (TO-BE)

- 수급/소모 경로 다양화
- 게임별 성장 요소로 장기 동기부여
- 단기(1회용 파워업) + 중기(게임별 업그레이드) + 장기(수집/만렙) 루프 구축

---

## 3. 재화 구조

### 3.1 Bagel Coin (기본 재화)

기존 재화를 그대로 사용. 수급/소모 경로만 확장.

#### 수급 경로

| 경로 | 설명 | 보상량 |
|------|------|--------|
| 게임 클리어 | 기존과 동일 (`baseReward + score x multiplier`) | 게임별 상이 |
| 최고기록 갱신 | 자기 bestScore 갱신 시 추가 보너스 | 20코인 |
| 마일스톤 달성 | 총 플레이 횟수/언락 수 기반 단계별 보상 | 50~500코인 |
| 일일 미션 (3개) | 매일 갱신되는 목표 달성 시 | 미션당 15~30코인 |

#### 소모 경로

| 경로 | 설명 | 비용 |
|------|------|------|
| 게임 언락 | 기존과 동일 (`unlockCost`) | 게임별 상이 |
| 1회용 파워업 구매 | 게임 시작 전 구매, 해당 판에만 적용 | 10~25코인 |
| 영구 업그레이드 구매 | 게임별 스탯 레벨업 (영구 적용) | 30~300코인 (레벨별 증가) |
| 이어하기 (Continue) | 게임오버 시 코인으로 부활 | 20코인 (횟수마다 2배) |

## 4. 1회용 파워업 (Consumable Power-Ups)

게임 시작 전 구매. 해당 판에서만 효과 적용. 매 판 소모.

### 4.1 공용 파워업 목록

| ID | 이름 | 효과 | 가격 | 적용 조건 |
|----|------|------|------|-----------|
| `time-extend` | 시간 연장 | 제한시간 +10초 | 15코인 | 시간제한 있는 게임 |
| `extra-life` | 목숨 추가 | 시작 시 목숨 +1 | 20코인 | 목숨 시스템 있는 게임 |
| `score-boost` | 스코어 부스트 | 획득 점수 x1.5 | 25코인 | 모든 게임 |
| `slow-motion` | 슬로우 모션 | 게임 속도 x0.8 (10초간) | 15코인 | 반응속도 게임 |
| `hint` | 힌트 | 정답/위치 1회 표시 | 10코인 | 퍼즐/퀴즈 게임 |

### 4.2 게임별 지원 파워업

각 미니게임 manifest에 `supportedPowerUps: PowerUpType[]` 필드 추가.
게임 시작 화면에서 해당 게임이 지원하는 파워업만 표시.

## 6. 이어하기 (Continue)

게임오버 시 코인을 소모하여 현재 상태에서 이어서 플레이.

| 이어하기 횟수 | 비용 |
|--------------|------|
| 1회차 | 20코인 |

| 3회차 | 80코인 |

## 9. 데이터 모델 변경

### 9.1 PlayerProgress 확장

```typescript
interface PlayerProgress {
  // 기존
  coins: number
  unlockedMiniGameIds: MiniGameId[]
  playCounts: Record<MiniGameId, number>
  bestScores: Record<MiniGameId, number>

  // 추가
  upgrades: Record<MiniGameId, Record<UpgradeType, number>>  // 게임별 업그레이드 레벨
  claimedMilestones: MilestoneId[]                           // 수령한 마일스톤
  dailyMissions: DailyMissionState                           // 일일 미션 상태
  totalCoinsEarned: number                                   // 총 획득 코인 (통계)
  totalCoinsSpent: number                                    // 총 소모 코인 (통계)
}
```

### 9.2 MiniGameManifest 확장

```typescript
interface MiniGameManifest {
  // 기존
  id: MiniGameId
  title: string
  description: string
  unlockCost: number
  baseReward: number
  scoreRewardMultiplier: number
  accentColor: string

  // 추가
  category: GameCategory                    // 카테고리 (A~H)
  supportedPowerUps: PowerUpType[]          // 지원하는 1회용 파워업
  supportsContinue: boolean                 // 이어하기 지원 여부
}
```

### 9.3 새로운 타입 정의

```typescript
type GameCategory =
  | 'time-limit'      // A: 시간제한형
  | 'reaction'        // B: 반응속도형
  | 'puzzle'          // C: 퍼즐형
  | 'runner'          // D: 러너/액션형
  | 'physics'         // E: 물리/조준형
  | 'versus'          // F: 대전/전략형
  | 'explore'         // G: 탐색형
  | 'rhythm'          // H: 음악/리듬형

type PowerUpType =
  | 'time-extend'
  | 'extra-life'
  | 'score-boost'
  | 'slow-motion'
  | 'hint'

type UpgradeType = 'upgrade-1' | 'upgrade-2' | 'upgrade-3'

// 카테고리별 업그레이드 의미는 런타임에서 해석
// 예: runner 카테고리의 upgrade-1 = 시작 속도 감소
```

## 10. 구현 우선순위

### Phase 1: 핵심 소모 경로 (즉시 효과)
1. 이어하기 (Continue) — 코인 소모 + 긴장감
2. 공용 1회용 파워업 5종 — 게임 시작 전 구매 UI

### Phase 2: 성장 시스템 (중기 동기부여)
3. 게임별 카테고리 분류 (A~H 매핑)
4. 카테고리별 영구 업그레이드 템플릿 구현
5. 업그레이드 구매 UI (게임 상세 화면)

### Phase 3: 수급 다양화 (장기 동기부여)
6. 마일스톤 시스템 + 보상 UI
7. 일일 미션 시스템 + 미션 UI
8. 최고기록 갱신 보너스

### Phase 4: 폴리싱
9. 코인 획득/소모 애니메이션
10. 업그레이드 레벨 표시 UI
11. 진행도 대시보드 (총 업그레이드 현황)
