- **모든 서브에이전트/팀 에이전트의 기본 모델은 `opus`** (`.claude/agents/` 설정 참고)

## Director 피드백 기반 게임 개발 원칙

### 화면 비율/레이아웃
- **게임 화면은 로비와 동일한 9:16 프레임(max-width: 432px) 안에서만 사용**. 전체화면(100vw)으로 늘어나면 안 됨
- `game-immersive` 모드에서도 참참참 같은 게임은 `max-width: 432px; margin: 0 auto`로 가운데 제한
- 좌우로 꽉 채우지 말고, 가운데 집중 레이아웃

### 캐릭터/UI 크기
- **캐릭터 이미지는 화면에서 압도적으로 크게** (최소 clamp(260px, 70vw, 380px) 이상)
- 텍스트도 크게 — 가이드 텍스트 2rem+, 결과 텍스트 2.5rem+, 점수 3rem+
- 버튼도 큼직하게 (min-height: 80px+)
- 캐릭터에 카드 프레임(border/background) 넣지 말 것 — 이미지만 깔끔하게
- **불필요한 UI 프레임/장식 최소화** — 과도한 테두리, 라운드 박스, 구분선 지양. 콘텐츠(캐릭터/텍스트)가 돋보여야 함

### 색상/배경
- **핑크 배경 금지** — 로비와 동일한 따뜻한 아이보리 톤 (#f5f4ef/#ede9df/#e8e5dc)
- 너무 어두운 배경(다크 네이비 등)도 안 됨 — 로비 분위기와 맞춰야 함
- 테두리/구분선은 로비와 동일한 #6b7280 그레이 계열

### 게임 연출
- 스프라이트 시트를 통으로 보여주지 말고, CSS `background-position`으로 잘라서 프레임별로 재생
- 캐릭터 둥둥 떠다니는 idle 애니메이션 불필요 — 상황별 애니메이션만 (hit/dodge/chant)
- 캐릭터 이름 표시 불필요, TURN 카운터 불필요
- 이펙트는 다양하고 찰지게 — 스파크 버스트, 스코어 팝, HP 비네팅, 화면 흔들림, 플래시 등 복합 사용
- **애니메이션은 최대한 다양하게** — 모든 상호작용(성공/실패/전환/콤보)에 고유한 애니메이션 부여
- **사운드/이펙트 적극 제작** — codeb로 상황별 효과음(히트/미스/콤보/전환/게임오버) 전부 생성. 무음 상태 없어야 함
- **설명 텍스트 최소화** — 캐릭터 이름, 턴 카운터, 부연 설명 등 플레이에 불필요한 텍스트는 빼기. 점수/콤보/HP 등 핵심 정보만 표시
- **이미지 생성 적극 활용** — 캐릭터, 아이콘, 배경, 이펙트, 손 스프라이트 등 가능한 모든 시각 요소를 codeb로 생성

### 게임 규칙 (참참참 기준)
- **수비 시 상대 방향을 미리 알려주지 않음** — 참참참 챈트가 끝난 후에야 방향 공개
- 공격 성공 시 상대 캐릭터가 바뀌어야 함 (매 턴 랜덤)
- 게임 템포는 빠르게 — resolve 800ms, role-switch 600ms, chant step 150ms

### 에셋 생성 (codeb)
- 기존 same-character 폴더의 김연자/박상민/박완규/서태지/송창식/태진아 픽셀 아트를 기본 캐릭터로 사용
- codeb로 생성한 이미지는 반드시 확인 후 적용 — 스프라이트 시트가 깨지면 원본 img 태그 방식으로 되돌릴 것
- 효과음은 codeb cg audio sfx/tts로 생성

### 새 게임 생성 시 필수 절차

- **게임 전용 아이콘은 반드시 codeb로 생성** — 로비 아이콘, 스테이지 캐릭터 이미지 모두 codeb cg image generate로 만들 것
- 기존 캐릭터 이미지를 아이콘 대용으로 쓰지 말 것

### codeb 에셋 품질 기준 (95점 룰)

codeb로 에셋(이미지/효과음/배경)을 생성할 때마다 **반드시 Read 도구로 결과물을 확인**하고 아래 5개 항목을 각각 100점 만점으로 자체 평가한다. **모든 항목이 95점 이상**이 될 때까지 프롬프트를 수정하여 재생성한다.

| 항목 | 평가 기준 |
|------|----------|
| **명확성** | 요청한 대상이 정확히 표현되었는가? 엉뚱한 요소가 없는가? |
| **픽셀아트 일관성** | 프로젝트의 기존 픽셀 아트 스타일과 일치하는가? |
| **배경 처리** | 투명 배경이 깔끔한가? 잔여 아티팩트가 없는가? |
| **해상도/선명도** | 충분히 선명하고, 게임 내 표시 크기에서 깨지지 않는가? |
| **용도 적합성** | 게임 내 사용 맥락(아이콘/캐릭터/이펙트/배경)에 적합한가? |

**절차:**

1. codeb로 에셋 생성
2. Read 도구로 결과물 확인 (이미지 직접 확인)
3. 5개 항목 자체 채점
4. 95점 미만 항목이 있으면 프롬프트 개선 후 재생성
5. 모든 항목 95점 이상이면 Director에게 결과 보고 후 적용
